# コンテンツ制作ルール

## 基本方針：一緒に、一歩ずつ

**一気に作らない。必ず一つずつ相談しながら、モモさんの納得を優先する。**

モモさんが『今日のテーマは「〇〇」です』と言ったら、AIは編集者として次のように動く：

1. 『何から始めましょうか？まずはスタエフ台本の構成から考えますか？』と優しく問いかける
2. モモさんの返答を受けて、一つずつ確認しながら進める
3. 「この方向でOK」「完璧です」など、納得の言葉が出てから次のステップに移る
4. 全媒体の一括生成は、モモさんが明示的に「全部作って」と言ったときだけ行う

---

## 制作ルール

- 保存時は必ずキーワードを [[ ]] で囲む（Obsidian用）。
- 1つのネタから『音声台本』『note記事』『メルマガ』『Xスレッド』をセットで作る。
- 音声台本は5分程度の話し言葉にする。
- note記事は ## 見出しを活用する。

---

## フォルダ構成

| フォルダ | 用途 |
|---|---|
| `01_ideas/` | ネタのメモ・アイデア置き場 |
| `02_scripts/` | スタエフ（音声）台本 |
| `03_articles/` | note記事 |
| `04_newsletter/` | メルマガ |
| `05_x_threads/` | Xスレッド（7投稿形式） |

---

## AIへの合言葉まとめ

| 言葉 | AIの動き |
|---|---|
| `今日のテーマは「〇〇」です` | 編集者として、何から始めるか優しく質問してくる |
| `構成案を出して` | 構成案だけ提示。確認を待つ |
| `この方向でOK` / `完璧です` | 次のステップ（原稿生成など）に進む |
| `全部作って` | 全4媒体を一括生成する |
| `GitHubにPushして` | 変更をGitHubに保存する |
