## スキル①｜一石四鳥・コンテンツ一括生成

### 何ができるか
[[ネタ]] を一つ決めるだけで、4種類の原稿をAIと一緒に作れる。

| 生成物 | 保存場所 |
|---|---|
| スタエフ（音声）台本 | `02_発信資産/02_scripts/` |
| [[note記事]] | `02_発信資産/03_articles/` |
| [[メルマガ]] | `02_発信資産/04_newsletter/` |
| [[Xスレッド]]（7投稿） | `02_発信資産/05_x_threads/` |

### 呼び出し方
```
今日のテーマは「〇〇」です
```
これだけ。AIが編集者として動き出す。

### AIの動き
『何から始めましょうか？まずはスタエフ台本の構成から考えますか？』と優しく聞いてくる。
一気に全部作らず、**モモさんの納得を確認しながら一歩ずつ**進める。

### 進め方の流れ
```
① 「今日のテーマは〇〇」と伝える
　↓
② AIが「何から始めますか？」と聞いてくる
　↓
③ 「スタエフ台本から」など答える
　↓
④ 構成案を見て「OKです」と言ったら原稿生成へ
　↓
⑤ 「全部作って」と言ったら4媒体を一括生成
```

### 一括生成したいときは
```
全部作って！
```
と言えば、4媒体を一気に生成してGitHubに保存するところまでやってくれる。

### ルールの場所
`02_発信資産/CLAUDE.md` に制作ルールと合言葉一覧が書いてある。

## スキル②｜Threads・愛の喝入れ投稿

### 何ができるか
[[思春期]]ママの心に刺さる、『共感×愛の喝入れ』スタイルのThreads投稿を常に【3パターン】作成できる。

**トーンのイメージ：** 親友が肩を掴んで言うような言葉。共感で受け止め、喝で背中を押す。

### 呼び出し方
```
この内容でThreads（スレッズ）を3パターン書いて
```

### AIの動き
秘伝の型（問いかけ → 共感 → 喝 → 希望）で、60文字前後・改行入りの投稿を3パターン一気に出してくる。

### 型のルール
| ルール | 内容 |
|---|---|
| 構成 | 問いかけ → 共感 → 喝 → 希望 |
| 文字数 | 1投稿60文字前後 |
| 形式 | 改行で間を作る |
| 禁止ワード | 『大丈夫』は一切使わない |
| 出力数 | 必ず3パターン |

## スキル③｜フォルダ整理・ファイル移動

### 何ができるか
フォルダの中身を別の場所に移動したり、構成を整理できる。

### どうやって呼び出すか
```
〇〇フォルダを△△の中に移動して。
```

### 注意点
- 移動前に「今の構成を教えて」と聞くと、Claudeが現状を確認してから動く
- 既存のフォルダと名前がかぶっていないか先に確認してもらうと安全

## スキル④｜GitHubへの保存（Push）

### 何ができるか
作ったファイルや変更を [[GitHub]] にアップロード（バックアップ）できる。
「いつ・何を・どう変えたか」が履歴として残る。

### どうやって呼び出すか
```
GitHubにPushして。
```
または、ファイル生成の指示の末尾に
```
終わったらGitHubにPushまでして。
```
と付けると一気に完結する。

### 保存先
- `momokiako` リポジトリ（GitHub上）
- URL: https://github.com/momoako2137-debug/momokiako

## スキル⑤｜新しいファイルの作成

### 何ができるか
指定したフォルダに、指定した内容のMarkdownファイルを新規作成できる。

### どうやって呼び出すか
```
〇〇フォルダの中に、△△.md というファイルを作って。
内容は〜〜〜。
```

### コツ
- Obsidian用に `[[キーワード]]` 形式にしてほしいときは「Obsidian用に」と添えるだけでOK
- 見出しを使ってほしいときは「## 見出しを使って」と指定する

## スキル⑥｜構成案の壁打ち

### 何ができるか
ネタや企画の方向性を一緒に考え、構成案を出してもらえる。
納得してから生成に進めるので、やり直しが少ない。

### どうやって呼び出すか
```
〇〇というテーマで発信したい。まず構成案を出して。
```

### コツ
- ターゲット・実際の仕組み・一番伝えたいことの3つを最初に伝えると、的外れな提案にならない
- 「完璧です」「この方向で」と言うと、Claudeが全文生成フェーズに移る

---

## スキル⑦｜リポジトリ作成・初回アップ

### 何ができるか
ローカルのフォルダを [[GitHub]] に新しいリポジトリとして登録し、初回アップロードできる。

### どうやって呼び出すか
```
このフォルダをGitHubに新しいリポジトリとして作成して、Pushして。
```

### 補足
GitHub Projects（カンバン）の作成には `project` スコープの権限が別途必要。
ターミナルで `! gh auth refresh -s project` を実行してから依頼するとOK。
